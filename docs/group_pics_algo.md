# 群照人物算法演进总结（v1~v4）

这篇文档用于对外分享我们的相册图库人物系统演进，不追踪细节调参，只讲主导思路如何一步步变化。

## v1：参考脸驱动的照片检索

### 核心目标

给定少量目标人物参考图，快速从候选照片里找出“包含这些人”的照片。

### 主流程

1. 为每个目标人物准备参考脸表示。
2. 对候选照片做人脸检测与 embedding 提取。
3. 候选脸与参考脸做相似度匹配，判断目标人物是否出现。
4. 按规则输出命中照片，并做简单分桶（如双人图/多人图）。

### 优劣分析

优点是上手快、心智简单，适合“临时找图”的短流程场景。  
真正的问题是它本质上是一次性检索：

- 强依赖参考图质量，参考图一旦偏，结果整体偏。
- 结果难沉淀为长期可维护的人物资产。
- 纠错基本是“本次任务修补”，无法自然反哺后续全库能力。

这直接引出下一版：系统需要从“找图工具”升级为“人物归属系统”。

## v2：人物原型驱动的全库归属

### 核心目标

把图库里每张人脸 observation 持续归属到“人物记录”，形成可维护、可纠错、可复用的人物库。

### 主流程

1. 照片入库并做人脸检测，生成 observation。
2. 提取 embedding，面向人物原型做候选召回与精排。
3. 对每张脸做自动归属、待复核或新人物候选判定。
4. 通过人工纠错回写归属，并更新人物原型。
5. 下游再基于归属结果做检索、导出等业务。

### 优劣分析

相较 v1，v2 完成了架构升级：从“临时检索”变成“可持续的人物系统”。  
但核心痛点也暴露出来：

- 自动归属仍偏激进，容易把错误样本写入人物。
- 原型更新过度依赖“现有归属全量样本”，误差会被放大。
- 用户需要持续做“排除式纠错”，长期成本高。

换句话说，v2 的方向对了，但“原型数据源”和“自动归属保守性”不够稳。  
这引出下一版：要先控制样本可信度，再让人物结构增长。

## v3：高信任样本池驱动的人物生长

### 核心目标

以“高信任样本池”为人物定义基础，降低误归属污染；同时把新人物发现纳入主流程，让人物库更稳地自动增长。

### 主流程

1. 先把“样本质量”与“身份相似度”分开建模。
2. 对未稳定归属样本先做聚类发现，形成可成长的人物候选。
3. 人物原型主要由高信任正样本构建，而非全量自动归属样本。
4. 自动归属更保守，人工确认更多承担“加法确认”而不是“减法清洗”。

### 优劣分析

v3 解决的是“系统可持续性”：  
宁可前期保守，也要避免错误样本进入原型后持续扩散。

这条路线的价值是：

- 人物定义更干净，系统越跑越稳。
- 新人物发现从辅助能力变为主流程能力。
- 用户工作流从“反复排错”转向“持续确认”。

真正暴露出来的主要问题，不是“这条路代价高”，而是它对质量评估调参的依赖度非常高，而且全流程联动下很难调稳：

- 质量门控决定了谁能进入高信任样本池，而质量分本身是多因子组合（面积、清晰度、姿态等），单个因子口径变化就会影响整体排序。
- 原始调参里，面积口径就经历过“面积占比 vs 绝对面积”的反复权衡：在超高清图库里，仅看占比会误伤不少可用小脸。
- 清晰度口径也不稳定：如果锐度计算过于直接，容易被 JPEG 噪声或背景纹理抬高，导致“看起来清晰分很高、实际不适合建模”的样本混入。
- 分位点归一化窗口（如 p10/p90 或更宽窗口）对不同批次数据很敏感，换一批图库后阈值可迁移性不强。
- 质量阈值一旦变动，会连锁影响后续发现、归属、原型更新，单点调优很容易“前面变好、后面变差”。
- 在完整链路里验证一轮调参成本高、反馈慢，导致参数收敛周期长，工程迭代效率不理想。
- 多轮调参实验后，对于高质量样本池的大小和内容，还是很不满意。

这轮复盘的结论是：  
v3 的方向没错，但要把“先把质量门调到非常完美”作为前置条件并不现实。  
这也是我们继续推进到 v4 的直接原因：先用两阶段聚类拿到更稳的结构，再在此基础上迭代质量与归属策略。

## v4：两阶段聚类驱动的人物归并

### 核心目标

以产品化落地为目标，把相册人物系统稳定在一条可持续链路上：  
`检测/对齐 -> embedding -> 聚类 -> 命名纠错 -> 增量并入`。  

v4 的重点不是“再做一版更复杂的单阶段聚类”，而是把第一阶段产出的高纯微簇进一步归并成人物，优先解决同一人被拆成多个簇的问题，让结果更适合直接进入产品闭环。

### 主流程

现状实现只做了 `检测/对齐 -> embedding -> 聚类` 这三步做想法验证，在工程执行上是三阶段串行：`detect -> embed -> cluster`。  
其中 `cluster` 阶段内部再拆成两段聚类：先做微簇（HDBSCAN），再做簇到人物的二次归并（AHC）。

1. 检测、对齐与样本落盘  
在人脸检测阶段，先按固定检测分辨率处理原图，再对每张脸落三类产物：局部人脸图、带框上下文图、对齐脸图。  
其中用于后续质量建模的基础量包括：

- 检测置信度：`det_confidence`。
- 人脸面积占比：`face_area_ratio = ((x2 - x1) * (y2 - y1)) / (W * H)`，其中 `W,H` 是原图宽高。
- 裁剪扩边比例：`pad_ratio = 0.25`，用于保留更多上下文。
- 上下文图最长边限制：`preview_max_side = 480`，用于控制展示与存储开销。

2. embedding 与质量分计算  
每张对齐脸输入 MagFace，得到原始向量 `f`（512 维），并计算：

- MagFace 质量量级：`magface_quality = ||f||2`。
- 归一化向量：`e = f / ||f||2`（当 `||f||2 <= 1e-9` 时直接使用 `f`，避免除零）。
- 综合质量分：  
  `quality_score = magface_quality * max(0.05, det_confidence) * sqrt(max(face_area_ratio, 1e-9))`。

这一步的作用是把“模型质量、检测可信度、可见面积”合成一个排序信号，为后续簇代表构建提供稳定依据。

3. 第一阶段微簇（HDBSCAN）  
对全部归一化 embedding 做 HDBSCAN，使用欧氏距离与 EOM 选簇策略，输出每张脸的簇标签与成员概率。

- `min_cluster_size = 3`
- `min_samples = 2`
- 当样本量小于 `max(2, min_cluster_size, min_samples)` 时，全部按噪声处理。
- 其中常量 `2` 是工程下限保护：确保单样本（`n=1`）不会进入 HDBSCAN 计算，避免边界场景报错。
- 标签 `-1` 视为噪声簇，不进入二次聚合。

4. 簇级表示构建（第二阶段输入）  
对每个微簇先按 `quality_score` 降序，取前 `K` 张高质量样本构建簇代表向量（`K = person_rep_top_k`，且实际 `K >= 1`）。

- 样本权重：`w_i = max(quality_score_i, 1e-3)`。
- 簇代表：`r = normalize(sum(w_i * e_i))`。

这一步把“face-level 离散点”压缩为“cluster-level 稳定节点”，降低低质量样本对二次聚类的扰动。

5. 候选边约束与距离矩阵构建  
先计算簇代表两两余弦距离，再施加结构约束，只保留可信候选边：

- 基础距离：`d(a,b) = 1 - clip(r_a · r_b, -1, 1)`。
- 近邻约束：先为每个簇按距离保留前 `k` 个近邻（`k = person_knn_k`，且有效值 `effective_k = max(1, min(k, n-1))`），再对候选边做对称化，取“双向近邻并集”。
- 因此单个簇在最终候选图中的连边数可能大于 `k`；`person_knn_k` 控制的是单侧取邻规模，而不是最终无向图里的严格度数上限。
- 同图冲突约束（可选，默认关闭）：若开启且两个簇含有同一张照片的人脸，则硬禁止合并。
- 对不满足约束的簇对，距离强制置为  
  `large_distance = max(person_merge_threshold + 1.0, 2.0)`，确保不会在阈值内被合并。

6. 第二阶段人物聚合（AHC）  
在约束后的簇间距离矩阵上执行层次聚合，按距离阈值切树得到人物分组。

- `linkage` 当前采用 `single`（支持 `average/single/complete`）。
- 合并阈值：`person_merge_threshold`（余弦距离阈值）。
- 输出是“人物 -> 多个微簇 -> 多张人脸”的层级结构，再进入命名、合并、拆分、忽略、增量并入的产品闭环。

当前全量图库运行参数为 `single + 0.26`；代码默认参数为 `single + 0.24`。

| 阈值/参数 | 当前全量值 | 代码默认值 | 含义 | 调大影响 | 调小影响 |
|---|---:|---:|---|---|---|
| `det_size` | 640 | 640 | 检测输入分辨率 | 小脸召回通常更好，但耗时和显存上升 | 速度更快，但小脸和边缘脸更易漏检 |
| `pad_ratio` | 0.25 | 0.25 | 人脸裁剪扩边比例 | 保留更多上下文，可能引入背景噪声 | 人脸更“紧”，但极端姿态时可能裁切过紧 |
| `preview_max_side` | 480 | 480 | 上下文图最长边 | 展示更清晰，存储与生成开销更高 | 生成更快更省空间，但 review 细节变少 |
| `max(0.05, det_confidence)` 的下限 `0.05` | 0.05 | 0.05 | 质量分中检测置信度地板值 | 地板更高会抬升低置信样本权重 | 地板更低会更强压制低置信样本 |
| `sqrt(max(face_area_ratio, 1e-9))` 的下限 `1e-9` | 1e-9 | 1e-9 | 质量分中面积项地板值 | 减小极小脸的惩罚，提升其参与度 | 更强抑制极小脸，质量分更保守 |
| `w_i = max(quality_score_i, 1e-3)` 的下限 `1e-3` | 1e-3 | 1e-3 | 构建簇代表时的最小权重 | 低质量样本影响力上升 | 低质量样本影响力下降，代表更“硬” |
| `min_cluster_size` | 3 | 3 | HDBSCAN 成簇最小规模 | 更偏保守，簇更纯但更碎、噪声增多 | 更容易成簇，召回上升但误聚风险增加 |
| `min_samples` | 2 | 2 | HDBSCAN 核心点密度要求 | 更偏保守，边界点更易成噪声 | 更宽松，簇更大但纯度可能下降 |
| 样本量下限常量 `2`（用于 `max(2, min_cluster_size, min_samples)`） | 2 | 2 | 防止单样本进入 HDBSCAN，作为稳定性保护阈值 | 提高该值会让小样本批次更早被整批判为噪声 | 降低到 1 会放开单样本路径，边界场景更易触发库内部异常 |
| `person_rep_top_k` | 3 | 3 | 每簇用于代表向量的高质量样本数 | 代表更平滑，抗偶然噪声更好，但可能被混入样本拉偏 | 代表更“尖锐”，对姿态变化鲁棒性下降 |
| `person_knn_k` | 8 | 8 | 二次聚类时每簇保留近邻数 | 候选合并边增多，召回提高但误并风险上升 | 候选边减少，误并降低但碎片化加重 |
| `person_enable_same_photo_cannot_link` | `false` | `false` | 是否开启同图冲突硬约束 | 开启后跨图安全性更高，但拼图/同图多次出现同人的召回会被压制 | 关闭后同图边界场景召回更好，但跨人误并风险上升 |
| `person_merge_threshold` | 0.26 | 0.24 | AHC 合并距离阈值（余弦距离） | 更激进，人物数减少，跨人误并风险上升 | 更保守，人物数增加，同人多簇更明显 |
| `person_linkage` | `single` | `single` | AHC 链接策略 | 切到 `average/complete` 通常更保守、更不易链式误并 | `single` 召回更高，但链式误并风险更高 |

### 优劣分析

v4 的价值在于，它把调研结论落成了可持续工程路径：  
在聚类子流程内先用 `MagFace + HDBSCAN` 获取高纯微簇，再用簇级二次聚类补足召回，形成“先保纯、再归并”的稳定策略。

二次聚类的阶段性取舍也更清晰：

- 当前主路径是 `AHC`，因为可解释、可调参、可快速接入产品流程。
- `linkage prediction` 作为中期升级方向，在召回上限受限时再投入。
- `FINCH/Chinese Whispers` 保留给更大规模数据场景。

这套方案的局限也很明确：  
同人拆分和跨人误并仍是持续博弈，`linkage` 与阈值对结果敏感，质量信号也需要随图库分布动态校准。

整体上，v4 不是终点，但它把系统推进到了一个可长期迭代的阶段：  
可以在统一框架下持续优化“纯度-召回”平衡，而不再回到单阶段方案反复拉扯。

### v4 优化记录：`min_samples=1 + person-consensus` 噪声回挂

这一节只记录当前实现上的一轮召回优化结果，不改上面的 v4 主流程描述。  
这轮优化聚焦第一阶段微簇召回，不讨论第二阶段人物归并阈值的进一步调整。

#### 调整内容

1. 第一阶段 HDBSCAN 继续保持 `min_cluster_size = 3`，仅把 `min_samples` 从 `2` 下调到 `1`。

    - 目标是放宽核心点密度要求，让一批“小而干净、但在 `min_samples = 2` 下被判成噪声”的样本先成微簇。

2. 在第一阶段 HDBSCAN 之后，增加 `person-consensus` 噪声回挂。

    - 先基于当前微簇做一轮 preliminary person 归并。
    - 仅对 `cluster_label = -1` 的噪声样本执行回挂。
    - 先为每个已有微簇构建代表向量，再按 person 收集候选微簇。
    - 若噪声脸到某个 person 内最佳微簇代表的余弦距离足够近，且相对第二候选有足够 margin，则把该脸挂回这个已有微簇。
    - 回挂不会生成新的 cluster，只会把噪声样本改挂到已有的 HDBSCAN 微簇 label 上。

#### 本轮参数

- 第一阶段微簇：
  - `min_cluster_size = 3`
  - `min_samples = 1`
- `person-consensus` 噪声回挂：
  - `person_consensus_distance_threshold = 0.24`
  - `person_consensus_margin_threshold = 0.04`
  - `person_consensus_rep_top_k = 3`

#### 本轮效果

- 基线目录：`.tmp/magface_hdbscan_review`
  - `noise_count = 1056`
  - `person_count = 96`
  - `person0 = 489`
  - `person1 = 340`
- 优化后目录：`.tmp/magface_hdbscan_review_min_samples_1_person_consensus_024`
  - `noise_count = 746`
  - `person_count = 113`
  - `person0 = 613`
  - `person1 = 459`
  - `person_consensus_attach_count = 147`

核心收益：

- 噪声数：`1056 -> 746`
- `person0`：`+124`
- `person1`：`+119`
- 多轮 diff review 结果显示，precision 基本未受影响

补充说明：

- 这里的 `person0 / person1` 指当前 review 页里按 `person_face_count / person_cluster_count / min_cluster_label` 排序后的临时编号，只用于描述这轮实验结果，不是稳定人物 ID。
- 这轮优化的本质是：先用更宽松的一阶段参数释放可成簇样本，再把一部分“已经明显接近已有 person 的噪声脸”挂回已有微簇，从而在基本不伤 precision 的前提下提升 recall。

### v4 TODO

1. 两个聚类阶段，都是 precesion 较高，但召回率仍有较大提升空间。
    - 算法流程看起来没大问题，但每个环节、每个阈值调参都要下功夫，但好在 AI 比我更知道怎么仿真、实验、分析。

2. 增量加图后，新样本会不会归入已有微簇，当前实现是：会一起重聚类，不是在线“挂到旧簇”。

    - cluster 阶段每次都读取当前库里所有 embedding 重新跑 HDBSCAN。
    - 所以新样本可能和旧样本落到同一簇，但这是“全量重算后的结果”，旧簇 ID 也可能变化。
    - 另外当前 detect 对已存在 face_id 的记录会把 embedding/cluster 清空，导致后续重算。
    - person_label/person_key 也完全不稳定。当前代码会先按 person_face_count/person_cluster_count/
     min_cluster_label 重排，再重新生成 person_{idx}，所以人物 ID 同样是一次运行内的临时编号，不能直接支撑后续命名或人工纠错沉淀。

3. det_size 太小了，现在绝大部分都是很高清的图，改成 1280 试试，检测速度不是大问题
