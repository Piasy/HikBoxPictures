# HikBox Pictures InsightFace 迁移设计文档

## 目标

将当前基于 `face_recognition` 的单参考图识别流程，升级为基于 `InsightFace + ArcFace` 的多参考图目录识别流程，同时保持工具的本地 CLI 形态、输出目录结构、Live Photo `MOV` 复制逻辑和时间元数据处理逻辑不变。

升级后的工具需要满足：

- `--ref-a-dir` 和 `--ref-b-dir` 分别接收一个参考图目录。
- 每个目录中的所有合法参考图都参与该人物的匹配。
- 候选图中的每张人脸都要分别计算它到 A、B 两组参考图的最小距离。
- 仍然要求同一张候选图中必须存在两张不同的人脸，分别命中 A 和 B，才算命中。

## 范围

范围内：

- 用 `InsightFace` 替换 `face_recognition` 作为检测与 embedding 引擎。
- 用参考图目录替换单张参考图输入。
- 在主 CLI、人脸裁剪脚本和距离调试脚本中统一使用同一套检测与距离计算逻辑；其中裁剪脚本只复用检测结果，不参与距离匹配。
- 保持 `only-two/YYYY-MM`、`group/YYYY-MM`、Live Photo `MOV` 复制和时间元数据保留行为不变。
- 更新安装脚本、README、测试与错误处理。

范围外：

- Apple Photos / PhotoKit 集成。
- 视频内容分析。
- GPU 推理配置。
- 参考图聚合、身份库持久化或多人物扩展到两人以上。
- 商业可再分发模型包的替换方案。

## 已选方案

采用“完全切换到 `InsightFace + ArcFace`，并将两组参考输入都改为目录”的方案。

关键决策：

- 新 CLI 参数使用 `--ref-a-dir` / `--ref-b-dir`，不复用旧的 `--ref-a` / `--ref-b`。
- 每个目录中的参考图不做均值聚合；目录级距离取该目录内所有参考 embedding 的最小距离。
- 首次运行允许 `InsightFace` 自动下载模型到本机缓存目录。
- 人脸裁剪脚本、距离调试脚本和主 CLI 共用同一套引擎边界，避免辅助脚本与主流程的检测结果长期漂移。

不选其他方案的原因：

- 不保留 `face_recognition` 兼容层，避免两套识别语义并存。
- 不拆成两阶段迁移，避免先做目录支持再做引擎替换带来的重复改造。
- 不额外引入引擎切换参数，保持 CLI 心智模型清晰。

## 运行时与依赖约束

- Python 依赖从 `face_recognition` / `dlib` 切换到 `insightface` 与 `onnxruntime`。
- `InsightFace` 的 Python 库代码可按 MIT 使用，但官方预训练模型包只允许非商业研究用途；该限制同样适用于自动下载的模型包和手动下载的模型包。该约束需要在 README 中明确提示。 
- 自 `insightface>=0.2` 起，推理后端为 `onnxruntime`；自 `insightface>=0.3.3` 起，`FaceAnalysis()` 初始化时会自动下载模型。默认使用 CPU 推理。 
- 默认模型包使用 `antelopev2`，因为它是官方模型包列表中的标准 face analysis 包，包含检测与识别所需模型。 

## CLI 接口

新 CLI 形态为：

```bash
hikbox-pictures \
  --input /path/to/photo-library \
  --ref-a-dir /path/to/person-a-dir \
  --ref-b-dir /path/to/person-b-dir \
  --output /path/to/output
```

接口约束：

- `--input` 和 `--output` 语义不变。
- `--ref-a-dir` / `--ref-b-dir` 必须存在且必须是目录。
- 旧参数 `--ref-a` / `--ref-b` 视为废弃并移除，不再兼容。
- 主 CLI 本次迁移不新增用户可调阈值参数；距离调试脚本继续保留阈值输入能力，用于分析而不是改变主流程契约。

## 用户流程

1. CLI 校验输入目录、两个人物参考目录和输出目录。
2. 初始化 `InsightFace` 分析引擎；如模型本地不存在，则触发自动下载。
3. 递归扫描两个人物参考目录中的受支持图片。
4. 对每张参考图做人脸检测与 embedding 提取。
5. 校验每张参考图都恰好包含一张脸。
6. 将每个人物目录中的所有合法参考 embedding 保留为一个列表。
7. 递归扫描输入目录中的候选图片。
8. 对每张候选图提取全部人脸与 embedding。
9. 计算候选图中每张脸到 A、B 两组参考 embedding 的最小距离。
10. 若存在两张不同候选脸分别命中 A 与 B，则判定候选图命中。
11. 按原有规则将命中结果导出到 `only-two/YYYY-MM` 或 `group/YYYY-MM`，并复制匹配的 Live Photo `MOV`。

## 架构

新增并调整如下模块边界：

### 1. `insightface_engine.py`

职责：

- 初始化 `FaceAnalysis`。
- 封装模型名、provider 和首次下载行为。
- 对外提供“从图片路径中提取检测结果”的统一接口。
- 返回检测框、embedding 和需要的调试信息。

这是本次迁移的关键边界。除该模块外，CLI、参考加载器、匹配器、调试脚本都不应直接依赖 `insightface` 细节。

### 2. `reference_loader.py`

职责从“单张参考图编码提取”调整为“参考目录加载”：

- 递归扫描参考目录。
- 过滤受支持图片扩展名。
- 逐张提取 embedding。
- 对任一参考图出现 `0` 张脸或 `>1` 张脸直接报错。
- 为每个人物返回一组参考 embedding 及其来源文件列表。

### 3. `matcher.py`

职责从单 embedding 匹配，升级为“候选脸对参考组的最小距离匹配”：

- 对每张候选图提取人脸与 embedding。
- 对每张候选脸分别计算到 A 组参考图的最小距离，以及到 B 组参考图的最小距离。
- 用同一套距离阈值规则为 A、B 生成候选命中集合。
- 仍要求两组命中集合能够对应到不同的人脸索引。
- 维持 `ONLY_TWO` / `GROUP` 分桶语义不变。

### 4. `cli.py`

职责保持编排层，但需要改成：

- 解析 `--ref-a-dir` / `--ref-b-dir`。
- 初始化引擎一次，在整个进程内复用。
- 调用参考目录加载与候选匹配的新接口。

### 5. `scripts/extract_faces.py`

职责保持“递归扫描并裁剪所有人脸”的辅助脚本定位，但检测实现切到新引擎：

- 复用 `insightface_engine.py` 的统一检测接口获取人脸框。
- 保持现有的方形外扩裁剪、越界补黑、输出尺寸和输出命名规则不变。
- 继续默认跳过输出目录，避免重复扫描自己生成的 PNG。
- 不引入 embedding 匹配逻辑，只消费检测框结果。

### 6. `scripts/inspect_distances.py`

职责保持调试脚本，但实现切到新引擎：

- 接收 `--ref-a-dir` / `--ref-b-dir`。
- 对候选图中每张人脸打印到 A/B 参考组的最小距离。
- 支持生成带人脸框和距离标注的临时图片。
- 默认跳过输入目录内的 `output/` 子目录，以及标注输出目录本身。

## 参考目录规则

参考目录的处理规则：

- 递归扫描目录中的 `HEIC`、`JPG`、`JPEG`、`PNG`。
- 忽略不支持的文件。
- 目录中至少要有一张可用参考图。
- 每张参考图必须恰好检测到一张脸。
- 参考目录内所有通过校验的图片都会被保留，不做聚合。

这样做的原因是：真实照片中的姿态、遮挡、口罩和光照差异很大，最小距离比“均值 embedding”更能保留多样性，避免平均后把有用的参考特征抹平。

## 匹配规则

候选图的匹配规则为：

- 对候选图中每张检测到的人脸，分别计算：
  - 它到 A 目录中所有参考 embedding 的最小距离。
  - 它到 B 目录中所有参考 embedding 的最小距离。
- 如果某张候选脸对 A 的最小距离满足阈值，则该脸记入 A 命中集合。
- 如果某张候选脸对 B 的最小距离满足阈值，则该脸记入 B 命中集合。
- 只有当 A、B 命中集合中存在不同的人脸索引时，这张候选图才算命中。

分类规则保持不变：

- 检测到人脸总数正好为 `2`，进入 `only-two`。
- 检测到人脸总数大于 `2`，进入 `group`。

设计约束：

- 距离阈值是引擎与匹配器内部的实现配置，不是当前主 CLI 的用户契约。
- 距离调试脚本必须打印原始距离，方便在样例图库上做阈值分析。

## 距离与 embedding 约定

- 识别模型使用 ArcFace embedding。
- 匹配器统一使用 InsightFace 返回的标准化 embedding 进行距离计算。
- 主 CLI 和调试脚本必须复用完全相同的距离计算函数，避免“调试脚本能看到的距离”和“主 CLI 实际命中的距离”不一致。

这里不将具体阈值写死为用户可见契约，是为了允许后续在不破坏 CLI 接口的前提下调优实现参数；但主 CLI 与调试脚本必须共享同一个默认值来源。

## 输出与导出行为

下列行为不变：

- 输出目录仍为：

```text
/output/
  only-two/
    YYYY-MM/
  group/
    YYYY-MM/
```

- 命中 `HEIC` 时仍查找并复制隐藏配对 `MOV`。
- 年月目录仍按 EXIF -> 文件创建时间 -> 文件修改时间解析。
- 导出时仍保留修改时间并尽力保留创建时间。
- 文件名冲突仍通过稳定后缀规避覆盖。

## 安装与首次运行行为

安装流程要同步调整：

- 安装脚本不再围绕 `dlib` 和 `setuptools<81` 提供兼容说明。
- README 改为说明 `insightface` / `onnxruntime` 依赖。
- README 明确写出“首次运行会自动下载模型，需联网，首次启动可能明显慢于后续运行”。
- README 明确写出预训练模型的非商业研究用途限制。

## 错误处理

致命错误：

- `--ref-a-dir` / `--ref-b-dir` 不存在或不是目录。
- 参考目录中没有任何可用图片。
- 任一参考图检测到 `0` 张脸。
- 任一参考图检测到多张脸。
- 引擎初始化失败。
- 模型自动下载失败或模型加载失败。

非致命错误：

- 候选图片解码失败。
- 候选图片未检测到人脸。
- 命中的 `HEIC` 缺失配对 `MOV`。
- 单张候选图片在推理过程中失败。

CLI 摘要维持现有统计结构：

- 扫描文件数。
- `only-two` 命中数。
- `group` 命中数。
- 解码失败数。
- 无人脸图片数。
- 缺失 Live Photo `MOV` 数。
- warning 列表。

## 测试策略

测试要整体迁移到新边界，不依赖真实 InsightFace 推理：

1. 参考目录加载
- 目录递归扫描受支持图片。
- 空目录失败。
- 参考图 `0 / 1 / 多张脸` 行为正确。

2. 引擎边界
- 引擎初始化参数正确。
- 候选图提取结果能稳定返回框与 embedding。
- 初始化或推理异常能被上层转化为明确错误。

3. 多参考图匹配
- 目录级匹配使用最小距离而不是均值。
- 候选图只有同一张脸同时贴近 A/B 时，不算命中。
- 候选图中两张不同脸分别命中 A/B 时，按人脸总数分桶。

4. CLI 编排
- 新参数 `--ref-a-dir` / `--ref-b-dir` 生效。
- 旧参数不再出现在 README 和测试构造中。
- 参考目录错误属于致命错误。

5. 距离调试脚本
- 使用目录输入。
- 打印目录级最小距离。
- 生成标注图时能输出框和距离。
- 默认跳过 `output/` 和标注输出目录。

6. 人脸裁剪脚本
- 使用新引擎做人脸检测。
- 外扩裁剪、补边、输出尺寸和命名规则保持不变。
- 默认跳过输出目录，避免递归处理生成结果。
- 无人脸图片和解码失败图片的统计行为保持清晰可观测。

7. 回归验证
- Live Photo 配对规则不变。
- 输出目录结构不变。
- 时间元数据回退顺序不变。

## 迁移影响

这次迁移是一个明确的 breaking change：

- CLI 输入参数从单图路径切换为目录路径。
- 安装依赖从 `face_recognition` / `dlib` 切换为 `insightface` / `onnxruntime`。
- `scripts/extract_faces.py` 的底层检测实现从 `face_recognition` 切换为 `InsightFace`。
- 调试脚本语义从“单参考图距离”切换为“参考目录最小距离”。

但以下用户可见行为保持稳定：

- 仍是一个本地 macOS CLI。
- 仍按 `only-two` / `group` 和 `YYYY-MM` 导出。
- 仍复制匹配的 Live Photo `MOV`。

## 风险与约束

- `InsightFace` 依赖链比当前方案更重，安装与环境问题可能集中出现在 `onnxruntime`、`opencv` 及其上游依赖上。
- 首次自动下载模型引入网络依赖，离线环境下首次运行会失败。
- 预训练模型包存在非商业研究用途限制，必须明确暴露给用户。
- 新引擎的距离尺度与当前 `face_recognition` 不同，旧阈值经验不可直接复用。
- 多参考图虽然能提升召回，但也会提高“任一参考图过宽导致误命中”的风险，因此需要把距离调试工具作为迁移的一部分保留。

## 成功标准

满足以下条件则视为本次迁移成功：

- 用户可以通过 `--ref-a-dir` / `--ref-b-dir` 提供两组参考图目录。
- 工具基于 `InsightFace + ArcFace` 在本地完成检测与匹配。
- 同一个人目录中的多张参考图会共同参与匹配，目录级距离取最小值。
- 主 CLI 与距离调试脚本使用同一套引擎和距离计算逻辑。
- `scripts/extract_faces.py` 与 `scripts/inspect_distances.py` 都复用新引擎边界，避免辅助脚本继续依赖旧框架。
- 输出结构、Live Photo 复制和时间元数据行为保持不变。
- 安装说明和首次运行说明覆盖模型自动下载与许可限制。
- 测试覆盖新的目录输入、引擎边界、多参考匹配与回归行为。
