# HikBox Pictures Template Matching 提升设计文档

## 目标

在继续使用 `DeepFace + ArcFace + retinaface + cosine + align` 基线的前提下，优先通过“参考图模板化、参考图清洗、轻量质量建模、阈值重标定”提升特定人物检索准确度，而不是直接引入 backbone 微调或新的训练链路。

本轮设计要解决的核心问题是：当前项目已经支持“多张参考图”，但匹配语义仍然是“候选脸到参考组的最小距离 + 单一阈值”，这对异常参考图、低质量抓拍和多人合照中的小脸都不够稳。

## 背景与现状

当前 DeepFace 版本的主流程如下：

1. 从 `ref-a-dir` / `ref-b-dir` 递归读取全部参考图。
2. 每张参考图要求且仅允许检测到一张脸。
3. 对候选图中的每张脸，分别计算它到 A、B 两组全部参考 embedding 的最小距离。
4. 只要 A、B 两组各有命中且落在两张不同的人脸上，就认为该照片命中。

现状的优点是简单、无状态、容易调试；但它有三个主要问题：

- 参考图中只要混入一张异常图，`min-distance` 就可能放大误报。
- 所有参考图对最终决策的贡献方式过于粗糙，没有区分清晰正脸和模糊侧脸。
- 主流程只有一个全局阈值，无法适配 A、B 两个人物在你的实际图库中的不同难度。

## 范围

范围内：

- 将“参考图列表”升级为“参考模板”。
- 为参考模板增加轻量的质量评分、离群样本清洗和聚合表示。
- 将候选脸到目标人物的匹配规则从“最小距离”升级为“模板分数”。
- 支持 A、B 各自独立的距离阈值，并提供阈值标定脚本。
- 在不改变 `only-two` / `group` 语义的前提下，增加照片级联合匹配分数，供调试和人工复核使用。
- 更新测试、README 与调试脚本。

范围外：

- 微调 `ArcFace` backbone 或引入任何新的识别训练流程。
- 引入新的重量级 quality-aware 网络或二阶段学习排序器。
- 引入持久化向量库、`DeepFace.find` 数据库模式或服务端部署形态。
- 修改输出目录结构、Live Photo 复制逻辑、时间元数据解析逻辑。

## 方案比较

### 方案 A：继续保留 `min-distance`，只增加更多参考图

优点：

- 改动最小。
- 与现有实现完全兼容。

缺点：

- 无法抑制脏参考图带来的误报。
- 仍然没有质量分层和人物级阈值能力。
- 继续依赖“偶然最接近的一张参考图”，稳定性不足。

### 方案 B：直接做模型后训练或微调

优点：

- 理论上有机会更贴近你的私有场景。

缺点：

- 工程成本高，依赖独立训练栈、标注数据和更长的验证周期。
- 数据量不足时收益不稳定，且容易把问题归因错到 backbone 上。
- 与当前本地 CLI 工具的维护边界不匹配。

### 方案 C：模板匹配基线增强

核心思路：

- 继续使用 DeepFace 产出的 embedding。
- 先把参考图清洗成更干净的模板，再用模板分数替代 `min-distance`。
- 用轻量启发式质量分数做聚合和排序。
- 在用户自己的图库上单独标定 A、B 的阈值。

优点：

- 与当前代码结构兼容，改动集中在模板构建、匹配和调试链路。
- 不需要训练新模型，交付速度快，便于逐步验证。
- 能直接改善异常参考图、参考图质量差异和全局阈值过粗的问题。

缺点：

- 仍然属于基于 embedding 的工程增强，不会从根本上改变 backbone 的能力上限。
- 质量分数是启发式而不是学习到的质量网络，上限有限。

本轮选择方案 C。

## 已选方案

### 1. 参考图从“embedding 列表”升级为“参考模板”

每个目标人物都由一个 `ReferenceTemplate` 表示，而不是一组裸 embedding。

模板包含以下信息：

- 原始参考样本列表。
- 清洗后保留的参考样本列表。
- 每张参考图的质量分数。
- 每张参考图到模板中心的距离。
- 模板中心向量（centroid embedding）。
- 模板匹配阈值。
- 调试所需的来源文件路径和清洗原因。

该设计保留“多样本 enrollment”的优势，同时把质量与清洗信息显式化，便于排查错误参考图。

### 2. 参考图质量分数采用轻量启发式，不引入新模型

本轮不新增训练依赖，质量分数完全由已有图片和检测框计算得到。

每张参考图的人脸质量分数由以下两项组成：

- `face_area_ratio`：人脸框面积占整张图面积的比例。
- `sharpness_score`：人脸裁剪区域灰度梯度方差，作为清晰度近似指标。

质量分数的计算约定：

- 对 `face_area_ratio` 和 `sharpness_score` 分别做稳健归一化。
- 默认使用 `0.6 * area_score + 0.4 * sharpness_score` 得到 `quality_score`。
- 若某项指标不可用，则退化为另一项；若两项都不可用，则回退到 `1.0`。

这样做的原因是：当前项目最常见的问题来自小脸和糊脸，这两项指标成本低、可解释性强，足以支撑第一轮基线增强。

### 3. 参考图清洗采用“中心距离排序 + 稳健离群阈值”

模板构建时，对每个人物的参考图执行以下流程：

1. 读取全部参考图并提取 embedding。
2. 对 embedding 做 `float32` 校验并保持当前单脸门槛不变。
3. 先计算一个临时中心向量。
4. 计算每张参考图到临时中心的距离。
5. 当参考图数量小于 `5` 时，不自动丢弃样本，只输出排序结果供用户检查。
6. 当参考图数量大于等于 `5` 时，使用 `median + 2.5 * MAD` 作为离群阈值。
7. 超过离群阈值的样本默认标记为剔除，但始终至少保留距离中心最近的 `3` 张图。

这套规则的意图是：

- 少量参考图时，避免自动清洗过度。
- 参考图足够多时，先把明显跑偏的样本剔除，减少误报源。

### 4. 聚合表示采用“保留成员 + 质量加权中心”双表示

模板不会只保留一个中心向量，而是同时保留：

- 清洗后的参考 embedding 列表。
- 一个质量加权后的中心向量。

中心向量的计算规则：

- 对每张保留样本使用 `quality_score` 作为权重。
- 对 embedding 做加权求和后再归一化，得到 `centroid_embedding`。

这样设计的原因是：

- 仅保留中心向量会丢失姿态和光照多样性。
- 仅保留成员列表又缺少一个稳定的模板整体表示。
- 双表示可以把“样本多样性”和“模板中心稳定性”同时保留下来。

### 5. 候选脸对人物模板的匹配规则改为 `top-k mean`

对候选图中的每张脸，分别计算它到 A、B 两个人物模板的模板分数。

模板分数定义为：

- 先计算该候选脸到模板内全部保留参考 embedding 的距离。
- 将距离从小到大排序。
- 取前 `k` 个距离的平均值，记为 `template_distance`。

`k` 的默认规则：

- `k = min(3, cleaned_reference_count)`。
- 当模板中只有 `1` 张或 `2` 张参考图时，自动退化到全部样本平均。

辅助诊断信息：

- 同时计算候选脸到 `centroid_embedding` 的距离，记为 `centroid_distance`。
- `centroid_distance` 本轮只用于调试输出和清洗分析，不直接参与命中判定。

不再默认使用 `min-distance` 的原因是：

- `min-distance` 对偶然最接近的一张脏参考图过于敏感。
- `top-k mean` 在参考样本数量有限的情况下更稳，且易于解释和调参。

### 6. 阈值从“全局一个”升级为“A/B 可分别设置”

主流程引入两级阈值来源：

- 若用户显式传入 A/B 独立阈值，则优先使用独立阈值。
- 否则回退到现有的全局 `distance_threshold`。
- 如果用户也未传全局阈值，则继续回退到 DeepFace 默认阈值。

这意味着：

- `DeepFaceEngine` 继续负责底层距离度量和默认阈值查询。
- 人物级模板负责持有自己的命中阈值。

### 7. 照片级判断仍保留 distinct faces 与 `only-two/group` 语义

候选照片是否命中的基础规则不变：

- A 和 B 必须分别命中两张不同的人脸。
- `only-two` / `group` 的归类继续沿用现有“大额外人脸”逻辑。

本轮只新增照片级联合分数，不改变落桶规则。

照片级联合分数定义为：

- 在所有满足“不同人脸索引”的 A/B 命中组合中，找到 `max(template_distance_a, template_distance_b)` 最小的一组。
- 该最小值记为 `joint_distance`。

`joint_distance` 的用途：

- 在调试脚本中展示“这张图为什么被判命中”。
- 在后续人工复核时提供一个统一的排序依据。
- 为以后扩展导出报告留下接口。

## CLI 与脚本设计

### 主 CLI

主 CLI 继续保留现有输入输出行为，并新增：

- `--distance-threshold-a`
- `--distance-threshold-b`

兼容规则：

- 若只传 `--distance-threshold`，则 A/B 共用该值。
- 若同时传入 A/B 独立阈值，则覆盖全局阈值。
- 若都未传，则继续使用 DeepFace 默认阈值。

主 CLI 内部改动：

- 从“加载 A/B embedding 列表”改为“构建 A/B 参考模板”。
- 匹配时调用模板分数计算逻辑，而不是 `min_distance`。
- 汇总信息保持现状，避免破坏现有使用方式。

### `scripts/inspect_distances.py`

调试脚本需要从“单纯打印最小距离”升级为打印模板诊断信息：

- A/B 模板保留样本数。
- A/B 模板被清洗掉的样本数与文件路径。
- 每张候选脸的 `template_distance_a`、`template_distance_b`。
- 每张候选脸的 `centroid_distance_a`、`centroid_distance_b`。
- 照片级 `joint_distance`。

### 新增 `scripts/calibrate_thresholds.py`

新增阈值标定脚本，用于在你自己的图库上得到更合适的阈值。

脚本输入：

- 一个目标人物模板目录。
- 一组正样本目录，要求“图片中存在该人物”。
- 一组负样本目录，要求“图片中不存在该人物”。
- 与主流程一致的 DeepFace 参数。

脚本行为：

1. 构建该人物的参考模板。
2. 扫描正样本目录中的照片，对每张照片取最佳命中脸的模板分数。
3. 扫描负样本目录中的照片，对每张照片取最危险误报脸的模板分数。
4. 扫描一组候选阈值，输出最佳 F1 阈值、最佳 Youden J 阈值和若干阈值分位点。
5. 输出建议传给主 CLI 的 `--distance-threshold-a` 或 `--distance-threshold-b`。

本轮不要求把阈值自动写回配置文件，先保证标定结果清晰可读。

## 模块边界

### 1. `deepface_engine.py`

保留为底层引擎边界，但职责聚焦为：

- 检测与 embedding 提取。
- 距离计算。
- 默认阈值查询。

不再让它直接承载人物级匹配阈值语义。

### 2. 新增 `reference_template.py`

负责：

- 参考图质量评分。
- 参考图清洗。
- 模板构建。
- 模板分数计算。

### 3. `matcher.py`

负责：

- 将候选图中的每张脸与 A/B 模板比对。
- 依据模板阈值形成 A/B 命中集合。
- 保留 distinct faces 与 `only-two/group` 分桶逻辑。
- 额外产出 `joint_distance` 和最佳配对索引作为调试信息。

### 4. `models.py`

新增或扩展的数据模型需要覆盖：

- 参考模板。
- 参考样本。
- 单脸模板比对结果。
- 照片级联合距离与最佳配对信息。

对外兼容性要求：

- `bucket` 语义不变。
- exporter 仍然只依赖 `PhotoEvaluation` 的最小必要字段。

## 测试策略

需要覆盖的核心测试包括：

- 参考图数量小于 `5` 时不会自动剔除样本。
- 参考图数量足够时，明显离群样本会被识别并剔除。
- `top-k mean` 的模板分数计算正确，且 `k` 会随参考图数量自动收缩。
- A/B 独立阈值会覆盖全局阈值。
- 匹配阶段仍然要求 A/B 是不同的人脸。
- `joint_distance` 会选择最佳 A/B 脸组合。
- `scripts/calibrate_thresholds.py` 能输出稳定、可解释的阈值建议。

## 风险与缓解

### 风险 1：参考图过少时自动清洗误伤

缓解：

- 参考图少于 `5` 张时只排序、不自动剔除。

### 风险 2：启发式质量分数偏向“大脸但不一定清晰”的图片

缓解：

- 同时引入清晰度指标。
- 质量分数仅用于权重和排序，不单独决定命中。

### 风险 3：用户未做阈值标定时效果提升有限

缓解：

- 保持 DeepFace 默认阈值回退链路可用。
- 提供专门标定脚本，降低标定门槛。

### 风险 4：设计范围膨胀到“训练新模型”

缓解：

- 本轮明确不进入训练栈。
- 将高阶 quality-aware comparator 和微调保留到后续阶段。

## 里程碑建议

本轮实现按以下顺序推进：

1. 先落地 `reference_template.py` 与 `matcher.py` 的模板匹配基线。
2. 再补 `inspect_distances.py` 的模板诊断输出。
3. 最后增加 `scripts/calibrate_thresholds.py` 与 README 文档。

这样可以先把主链路效果跑起来，再补调试与标定工具，减少一次性改动面。

## 与既有设计文档的关系

本设计文档建立在 `2026-04-10-hikbox-pictures-deepface-design.md` 之上，但会修改其中两条关键假设：

- 参考图不再只是“全部 embedding 的最小距离”语义，而是升级为模板语义。
- 距离阈值不再默认只能全局共享，而是支持人物级标定。

其余约束继续保持不变：

- 仍使用 DeepFace 作为底层检测与 embedding 来源。
- 仍保持本地 CLI、目录扫描、导出结构和 Live Photo 复制逻辑不变。
